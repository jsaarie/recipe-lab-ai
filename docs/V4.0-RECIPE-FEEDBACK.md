# v4.0 — Recipe Feedback

## Overview

Allow users to capture feedback after completing a recipe. Users can rate recipes with stars (1–5) and add plain-text cook notes. Feedback is tied to the user's account and stored alongside saved recipes.

## Requirements

### Feedback Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| Rating | 1–5 stars | No | Star rating for the recipe |
| Cook Notes | Plain text | No | Free-text notes about the cook |
| Created At | Timestamp | Auto | When feedback was first added |
| Updated At | Timestamp | Auto | When feedback was last edited |

### Entry Points

1. **Post-Lab Completion** — After the user completes the final step in Lab View, a button appears (e.g., "Rate This Recipe") that opens a feedback modal overlay.
2. **Library** — Users can add or edit feedback from any saved recipe in their Library.

### Behavior

- **Auth required** — Only logged-in users can leave feedback. Feedback is stored on the user's `savedRecipes` document in MongoDB.
- **Editable** — Users can update their rating and notes at any time.
- **Optional** — Both fields are optional. Users can rate without notes or add notes without a rating.
- **No reminders** — If the user skips the post-lab prompt, there is no badge or reminder. They can always add feedback later from the Library.

### UX

- **Post-Lab Modal** — A centered modal overlay triggered by a button on the lab completion screen. Contains star selector and text area.
- **Library Cards** — Star rating is displayed on recipe cards in the Library grid (if rated).
- **Recipe Detail** — Full feedback (stars + notes) visible when opening a recipe from the Library.

### Data Model

Feedback fields are added to the existing `savedRecipes` document:

```ts
interface RecipeFeedback {
  rating?: number        // 1–5
  cookNotes?: string     // plain text
  feedbackCreatedAt?: Date
  feedbackUpdatedAt?: Date
}
```

### API

- `PATCH /api/recipes/[id]/feedback` — Create or update feedback (rating, cookNotes)
- `GET /api/recipes/[id]` — Returns recipe with feedback fields included
